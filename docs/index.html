When Point Metrics Mislead

Structure-Aware Evaluation Reveals Conditional Ranking Shifts in Time Series Anomaly Detection

Youngmin Ko

Point-wise metrics can change time-series anomaly detection rankings when benchmark anomalies are sustained temporal segments rather than isolated spikes.

arXiv: Coming Soon Code / Artifact BibTeX

Headline Evidence

14/60 deep-model rank flips
44/126 rank flips with classical baselines
0 short anomaly segments in four industrial benchmarks
4,498 TSB-AD-M recomputed rows

Key Findings

Metric disagreement is structured

AUC-ROC and Aff-F1 disagree in pairwise rankings, and that disagreement is systematic rather than random noise.

Dataset structure matters

When anomalies are long segments, timestamp-level metrics and segment-level metrics emphasize different behaviors.

SAEScore is a reporting composite

SAEScore is presented as a transparent reporting blend, not as a universal replacement leaderboard metric.

Reproducibility artifact included

The repository ships derived outputs, validation scripts, and lightweight tests for direct verification.

Problem -> Finding -> Artifact

Problem

Point metrics evaluate timestamp discrimination, which can underrepresent continuity and cohesion of anomaly segments.

Mismatch

Industrial anomaly benchmarks often contain sustained temporal events rather than isolated point spikes.

Result

Model rankings can shift under segment-aware reporting, motivating structure-aware evaluation for audit-quality analysis.
